## 2. Google Multi-Agent 模式映射

基于 Google ADK 8种 Multi-Agent 设计模式，映射到本系统：

| #    | Google 模式                   | 本系统应用位置                                         | 触发条件       |
| ---- | ----------------------------- | ------------------------------------------------------ | -------------- |
| 1    | **Sequential Pipeline**       | 主工作流 intake→design→plan→code→review→test→archive   | 所有任务       |
| 2    | **Parallel / Fan-out**        | Review 阶段：Claude Code 实现 ∥ Codex 准备 Review 环境 | 代码提交后     |
| 3    | **Router / Dispatcher**       | 判断简单工作流 vs 复杂 Agents Team                     | 任务评估后     |
| 4    | **Human-in-the-Loop**         | requirements 确认 / design 批准 / 生产部署审批         | 每个检查点     |
| 5    | **Critic / Self-Review**      | Codex 审查 Claude Code 输出，不通过则打回修改          | 每个任务完成后 |
| 6    | **Orchestrator + Sub-Agents** | 复杂任务：Claude Code 主导 + N 个 Codex 子代理并行     | 评估为复杂     |
| 7    | **Hierarchical Multi-Agent**  | TG Gateway → Router → Phase Agents → Task Agents       | 系统全局       |
| 8    | **Shared State**              | specs/ 目录作为共享状态，所有 agent 读写同一文件       | 始终           |

```
Pattern #3 Router 判断逻辑:
  if 预估 > 4h OR 涉及文件 > 20 OR 跨服务 > 2:
    → Agents Team (Pattern #6)
  elif 预估 < 30min:
    → Light Workflow (只走 propose + apply)
  else:
    → Standard Workflow (完整 5 阶段)
```

## 4. 核心工作流

### 主流程 (Standard Workflow)

```
[TG 消息 / Jira URL / 手动]
         │
         ▼
  SKILL: 00-router
  ┌─ 评估任务规模 ─────────────────────────────────────┐
  │ < 30min → Light Flow                              │
  │ 30min-4h → Standard Flow                         │
  │ > 4h / 复杂 → Agents Team                        │
  └───────────────────────────────────────────────────┘
         │
         ▼ Standard Flow
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  PHASE 1: INTAKE (SKILL: 01-intake)
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  1. 解析输入 (TG文本 / Jira API / 手动描述)
  2. 生成 specs/<feature>/v1/requirements.md
  3. ⏸ 人工确认 → TG 发送确认请求
  4. 等待 "✅ 确认" 或 "❌ 修改"
         │
         ▼ 确认后
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  PHASE 2: DESIGN (SKILL: 02-design)
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  1. 读取 docs/ARCHITECTURE.md
  2. 读取 docs/SECURITY.md
  3. 读取 docs/CODING_GUIDELINES.md
  4. 读取 .claude/CLAUDE.md
  5. 结合 requirements.md 生成 design.md
     - 架构决策 / API 设计 / DB Schema / 安全考量
  6. ⏸ 人工确认设计文档
         │
         ▼ 确认后
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  PHASE 3: PLAN (SKILL: 03-plan)
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  1. 读取 design.md 拆解任务
  2. 生成 tasks.md (每任务 ≤ 30min, 含文件路径)
  3. 识别可并行任务 → 标注 [PARALLEL]
  4. 识别复杂子模块 → 标注 [SUBAGENT]
  5. ⏸ 人工确认任务计划
         │
         ▼ 确认后
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  PHASE 4: IMPLEMENT (SKILL: 04-implement)
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  1. git worktree 创建隔离分支
  2. 逐任务执行 TDD 循环:
     🔴 RED   → 写失败测试
     🟢 GREEN → 最少代码通过
     ♻️ REFACTOR → 重构
  3. 每任务完成 → 触发 SKILL: 05-review
         │
         ▼
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  PHASE 5: REVIEW (SKILL: 05-review)
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  [Parallel: Google Pattern #2]
  ├── Claude Code: 继续下一任务
  └── Codex (OpenCLI): 审查已完成任务
       Stage 1: 规格合规性审查 (对照 requirements.md)
       Stage 2: 代码质量审查 (安全/性能/可维护性)
       → PASS: 继续
       → FAIL: 打回 Claude Code 修改
         │
         ▼ 全部任务完成
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  PHASE 6: TESTING (SKILL: 06-testing + 07-e2e-web-mcp)
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  [由 Codex 执行]
  1. 单元测试: Jest / Vitest
  2. 集成测试: Comet / Atlas
  3. E2E 测试:
     - Playwright 传统模式
     - Google Web MCP Remote Debug 模式 (推荐)
  4. 生成测试报告 → TG 发送报告摘要
  5. 覆盖率 < 阈值 → 打回补写测试
         │
         ▼ 全部通过
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  PHASE 7: ARCHIVE (SKILL: 08-archive)
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  1. 更新 README.md (功能说明)
  2. 更新 docs/ARCHITECTURE.md (架构变动)
  3. 更新 docs/SECURITY.md (安全变动)
  4. 更新 docs/CODING_GUIDELINES.md (规范变动)
  5. 更新 CHANGELOG.md
  6. specs 版本迭代 (v1 → v2...)
  7. TG 发送完成通知 + 测试报告链接
  8. ⏸ 人工审批 → 合并 PR / 部署

### `.claude/skills/00-router/SKILL.md`

```markdown
# Skill: Router — 任务路由判断

## 触发时机
收到任何新任务描述时，MUST 首先执行本 Skill

## 执行步骤

### Step 1: 解析任务描述
从 TG 消息 / Jira URL / 手动描述中提取：
- 功能范围描述
- 预估影响的文件数量
- 是否跨服务/跨模块
- 是否有明确截止时间

### Step 2: 评估路由
```

评估标准: COMPLEXITY_SCORE = 0

if 预估工时 < 30min:     COMPLEXITY_SCORE = 1 if 预估工时 30min-2h:    COMPLEXITY_SCORE = 2 if 预估工时 2h-4h:       COMPLEXITY_SCORE = 3 if 预估工时 > 4h:        COMPLEXITY_SCORE = 4

if 涉及文件 > 20:        COMPLEXITY_SCORE += 2 if 跨服务 > 1:           COMPLEXITY_SCORE += 2 if 涉及安全/支付/认证:    COMPLEXITY_SCORE += 3 if 涉及数据库schema变更:  COMPLEXITY_SCORE += 2 if 依赖外部API > 2:      COMPLEXITY_SCORE += 1

路由决策: SCORE = 1          → LIGHT_FLOW (只用 01+04+08) SCORE = 2-4        → STANDARD_FLOW (完整7阶段) SCORE >= 5         → AGENTS_TEAM (读取 09-agents-team/SKILL.md)

```
### Step 3: 初始化 Spec 目录
```bash
# 将需求用英文命名(kebab-case)
FEATURE_NAME="<english-feature-name>"
VERSION="v1"

mkdir -p specs/${FEATURE_NAME}/${VERSION}
touch specs/${FEATURE_NAME}/${VERSION}/requirements.md
touch specs/${FEATURE_NAME}/${VERSION}/design.md
touch specs/${FEATURE_NAME}/${VERSION}/tasks.md

echo "路由完成 → 进入: [LIGHT/STANDARD/AGENTS_TEAM]"
echo "Spec 目录: specs/${FEATURE_NAME}/${VERSION}/"
```

### Step 4: TG 通知

调用 SKILL: 10-tg-notify，发送： "📋 任务已接收\n功能: {FEATURE_NAME}\n路由: {FLOW_TYPE}\n评分: {SCORE}"

```

3. 运行测试，确认通过

1. 在保持测试通过的前提下优化代码
2. 消除重复 (DRY)
3. 提高可读性
4. 再次运行测试，确认仍然通过

## 7. Telegram Bot 驱动层

### `tg-gateway/bot.py`

```python
#!/usr/bin/env python3
"""
Telegram Bot Gateway — AI 虚拟办公室接入层
"""

import os
import asyncio
import subprocess
from telegram import Update, InlineKeyboardButton, InlineKeyboardMarkup
from telegram.ext import (
    Application, CommandHandler, MessageHandler, 
    CallbackQueryHandler, filters, ContextTypes
)

BOT_TOKEN = os.environ["TG_BOT_TOKEN"]
ALLOWED_CHAT_IDS = os.environ["TG_ALLOWED_CHATS"].split(",")
CLAUDE_CODE_PATH = os.environ.get("CLAUDE_CODE_PATH", "claude")
PROJECT_PATH = os.environ["PROJECT_PATH"]

# ──────────────────────────────────────────────────────────────────────
# 消息处理器
# ──────────────────────────────────────────────────────────────────────

async def handle_new_task(update: Update, context: ContextTypes.DEFAULT_TYPE):
    """处理新任务请求 (自然语言描述)"""
    if str(update.effective_chat.id) not in ALLOWED_CHAT_IDS:
        return
    
    task_description = update.message.text
    
    # 检测是否是 Jira URL
    if "jira" in task_description.lower() or "atlassian" in task_description:
        await update.message.reply_text("🔍 检测到 Jira URL，正在读取需求...")
        task_description = await fetch_jira_issue(task_description)
    
    await update.message.reply_text(
        f"📥 *任务已接收*\n\n```\n{task_description[:200]}...\n```\n\n"
        f"正在启动 Claude Code 分析任务规模...",
        parse_mode="Markdown"
    )
    
    # 异步启动 Claude Code
    asyncio.create_task(
        run_claude_code(task_description, update.effective_chat.id)
    )

async def handle_approve(update: Update, context: ContextTypes.DEFAULT_TYPE):
    """处理 /approve 命令"""
    feature = context.args[0] if context.args else None
    if not feature:
        await update.message.reply_text("用法: /approve <feature-name>")
        return
    
    # 写入审批信号文件
    signal_file = f"{PROJECT_PATH}/.claude/signals/{feature}.approve"
    os.makedirs(os.path.dirname(signal_file), exist_ok=True)
    with open(signal_file, "w") as f:
        f.write(f"approved by {update.effective_user.username}")
    
    await update.message.reply_text(f"✅ *{feature}* 已批准，继续执行...", parse_mode="Markdown")

async def handle_reject(update: Update, context: ContextTypes.DEFAULT_TYPE):
    """处理 /reject 命令"""
    if len(context.args) < 2:
        await update.message.reply_text("用法: /reject <feature-name> <原因>")
        return
    
    feature = context.args[0]
    reason = " ".join(context.args[1:])
    
    signal_file = f"{PROJECT_PATH}/.claude/signals/{feature}.reject"
    with open(signal_file, "w") as f:
        f.write(reason)
    
    await update.message.reply_text(
        f"❌ *{feature}* 已打回\n原因: {reason}", 
        parse_mode="Markdown"
    )

async def handle_status(update: Update, context: ContextTypes.DEFAULT_TYPE):
    """查询当前工作状态"""
    # 扫描 specs/ 目录中未完成的任务
    result = subprocess.run(
        ["grep", "-r", r"\[ \]", f"{PROJECT_PATH}/specs/"],
        capture_output=True, text=True
    )
    
    if result.stdout:
        await update.message.reply_text(
            f"📊 *当前未完成任务*:\n```\n{result.stdout[:800]}\n```",
            parse_mode="Markdown"
        )
    else:
        await update.message.reply_text("✅ 所有任务已完成!")

async def handle_hotfix(update: Update, context: ContextTypes.DEFAULT_TYPE):
    """触发紧急修复流程"""
    description = " ".join(context.args) if context.args else update.message.text
    
    await update.message.reply_text(
        f"🚨 *紧急修复模式*\n\n问题: {description}\n\n"
        f"正在绕过常规流程，启动快速修复...",
        parse_mode="Markdown"
    )
    
    asyncio.create_task(
        run_claude_code(f"[HOTFIX] {description}", update.effective_chat.id)
    )

# ──────────────────────────────────────────────────────────────────────
# Claude Code 调用
# ──────────────────────────────────────────────────────────────────────

async def run_claude_code(task: str, chat_id: str):
    """通过 subprocess 调用 Claude Code"""
    
    # 构建 Claude Code 命令
    # Claude Code 将读取 .claude/CLAUDE.md 和 Skills
    prompt = f"""
你收到了一个新任务。请严格按照 .claude/skills/00-router/SKILL.md 开始执行。

任务描述:
{task}

记住:
1. 先读取 .claude/CLAUDE.md
2. 然后执行 00-router/SKILL.md 中的路由判断
3. 通过 scripts/notify-tg.sh 发送进度通知到 chat_id: {chat_id}
4. 等待 TG 的 /approve 或 /reject 信号 (检查 .claude/signals/ 目录)
"""
    
    process = await asyncio.create_subprocess_exec(
        CLAUDE_CODE_PATH, "--print", prompt,
        cwd=PROJECT_PATH,
        stdout=asyncio.subprocess.PIPE,
        stderr=asyncio.subprocess.PIPE
    )
    
    stdout, stderr = await process.communicate()
    
    if process.returncode != 0:
        # 通知错误
        subprocess.run([
            "./scripts/notify-tg.sh",
            f"🚨 Claude Code 执行错误\n{stderr.decode()[:500]}"
        ], cwd=PROJECT_PATH)

async def fetch_jira_issue(url: str) -> str:
    """从 Jira URL 获取需求描述"""
    import re
    import httpx
    
    # 提取 Issue Key
    match = re.search(r'browse/([A-Z]+-\d+)', url)
    if not match:
        return url
    
    issue_key = match.group(1)
    jira_base = os.environ.get("JIRA_BASE_URL", "")
    jira_token = os.environ.get("JIRA_TOKEN", "")
    
    async with httpx.AsyncClient() as client:
        response = await client.get(
            f"{jira_base}/rest/api/3/issue/{issue_key}",
            headers={"Authorization": f"Bearer {jira_token}"}
        )
        data = response.json()
        
        summary = data["fields"]["summary"]
        description = data["fields"].get("description", {})
        # 简化 Atlassian Document Format
        desc_text = str(description)[:500]
        
        return f"Jira {issue_key}: {summary}\n\n{desc_text}"

# ──────────────────────────────────────────────────────────────────────
# 主程序
# ──────────────────────────────────────────────────────────────────────

def main():
    app = Application.builder().token(BOT_TOKEN).build()
    
    # 命令处理器
    app.add_handler(CommandHandler("approve", handle_approve))
    app.add_handler(CommandHandler("reject", handle_reject))
    app.add_handler(CommandHandler("status", handle_status))
    app.add_handler(CommandHandler("hotfix", handle_hotfix))
    app.add_handler(CommandHandler("pause", lambda u, c: None))  # TODO
    app.add_handler(CommandHandler("resume", lambda u, c: None))  # TODO
    
    # 消息处理器 (非命令的普通消息视为新任务)
    app.add_handler(MessageHandler(
        filters.TEXT & ~filters.COMMAND, 
        handle_new_task
    ))
    
    print("🤖 AI 虚拟办公室 TG Bot 启动中...")
    app.run_polling(allowed_updates=Update.ALL_TYPES)

if __name__ == "__main__":
    main()
```
